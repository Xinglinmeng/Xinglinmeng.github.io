Github个人博客与Hexo

使用GitHub Pages 搭建 Hexo 静态博客网站优点：免费！！！！！

一、介绍

Github Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，可以在 GitHub 仓库里托管和发布自己的静态网站页面。

Hexo 是一个快速、简洁且高效的静态博客框架，它基于Node.js 运行，可以将撰写的Markdown 文档解析渲染成静态的 HTML 网页。

在本地撰写 Markdown 格式文章后，通过 Hexo 解析文档，渲染生成具有主题样式的 HTML 静态网页，再推送到 GitHub 上完成博文的发布。

二、环境

注册Github个人账户、创建用户名仓库(之后可以访问https://用户名.github.io/地址来访问博客)

安装Node.js(使用npm（切换国内镜像源即可），建议使用cnpm）

安装Git(右键文件夹映出菜单使用Git Bash)

使用cnpm安装Hexo并创建文章、修改主题、部署到github等

三、Node.js

或者直接安装nvm用来管理版本会方便一些

  1. 同意许可

  2. 自定义安装目录

  3. 默认（Node.js Runtime），next

  4. install

  5. win键+R输入cmd打开命令提示符

    C:\Users\xlm>node -v
    v18.20.3
    C:\Users\xlm>npm -v
    10.7.0

  6. 安装cnpm，设置包的下载源

    npm install -g cnpm --registry=https://registry.npmmirror.com

  7. 或者不安装cnpm

    在安装目录下创建node_cache还和node_global文件夹

    npm config set prefix "D:\nodejs\node_global"
    npm config set cache "D:\nodejs\node_cache"

    配置镜像站

    npm config set registry https://registry.npmmirror.com

    输入命令npm config list 显示所有配置信息，C:\Users\Administrator.npmrc；会显示之前配置的文件夹

    由于目录改变，需要修改环境变量（我的电脑右键-高级属性-高级-环境变量-系统变量新建）：NODE_PATH=D:\nodejs\node_global\node_modules

四、Git

安装过程一般都是默认配置（安装目录自定义），直接next；之后可以在开始菜单中找到git bash

Git 需要设置用户名和邮箱作为一个标识，在命令行输入： 

git config --global user.name "user_name" #GitHub 用户名
git config --global user.email "user_email" #GitHub 注册的邮箱

创建SSH密钥（在C:\Users\用户名目录\ .ssh目录生成、其中的id_rsa.pub 密钥SSH连接会用到）

ssh-keygen -t rsa -C "user.email" # user.email为GitHub上注册的邮箱

在github配置完密钥后

#检测设置成功？
ssh -T git@github.com

五、Github

访问官网“https://github.com/”，注册账户（邮箱+密码），若访问慢可下载Watt Toolkit（在git与github连接操作时不建议使用，会强制修改文件的github地址为本地，导致访问被拒绝）

创建仓库（格式为用户名.github.io）

配置SSH密钥：点击Github头像->Settings->SSH and GPG keys->New SSH key（将pub文件内容填入）

六、Hexo

安装 hexo 博客文档框架（想在哪个文件夹创项目，就用git bash打开。-g代表全局）

cnpm install hexo-cli

创建项目 blog 并初始化

hexo init blog
cd hexo-blog
npm install
hexo g # 生成静态文件
hexo s # 开启本地预览	浏览器访问 http://localhost:4000

部署到github

配置站点配置文件

repo可以在github仓库的Code的SSH复制（E:\Blogs\ _config.yml）

deploy:
  type: git
  repo: git@github.com:Xinglinmeng/Xinglinmeng.github.io.git
  branch: master
npm install hexo-deployer-git --save # 安装部署插件
hexo d #部署

问题：若hexo d执行后，github不更新

hexo clean && hexo generate
hexo deploy
